② 適用は「施行日以降に提起された訴え」

  • 令和8年5月20日以前に提起された事件（旧法事件）は従来どおり書面・FAX

③ 1週間でみなし送達

  • 裁判所からのシステム送達は、未閲覧でも1週間経過で送達の効力発生（民訴法109条の3）
  • そこから上訴期間等が起算される。メール不達でも進行する

  • 納付を怠ると訴状却下命令のリスク（民訴法137条の2）

  • パスワード付きPDF不可、ファイル名は100文字以内

⑥ アップロードしたら取り消せない

  • 提出後の差替え・削除は裁判所しかできない。誤提出は担当書記官に電話
  • 提出前のプレビュー確認を習慣に

  • 初回の閲覧・ダウンロード・印刷で「アクセス状況の記録」が付き、全当事者にメール通知される

⑧ 受領書・押印は新法事件では不要

  • 新法事件は受領書提出不要（アクセス記録が代替）。書面の押印も不要
  • 旧法事件では従来どおり受領書を提出

第1章義務化の全体像

1.1 何が義務になったか

令和8年5月21日、民事裁判手続のデジタル化（いわゆるフェーズ3）に係る改正民事訴訟法が全面施行されました。これにより、民事訴訟における申立てその他の申述（申立て等）は、誰でもインターネット経由（mints）で行うことが可能になり（民訴法132条の10）、次の者には mints による提出が義務付けられました（民訴法132条の11第1項。条文全文は第12章）。

義務の対象者対象となる事件
委任を受けた訴訟代理人（弁護士等。民訴法54条1項ただし書の許可代理人は除く）当該委任を受けた事件
法務大臣権限法による指定を受けた者（国の指定代理人）当該指定の対象事件
地方自治法153条1項の委任を受けた職員当該委任を受けた事件

義務の内容は2つあります。

  • オンライン提出義務：訴状・準備書面・証拠等の申立て等を mints で提出しなければならない（口頭ですることができる申立てを口頭でする場合を除く）。
  • システム送達届出義務：システム送達（後記1.3）を受ける旨の届出をしなければならない（132条の11第2項）。
⚠ 例外 ― システム障害時 裁判所の電子計算機の故障その他本人の責めに帰することができない事由により mints で提出できない場合には、義務は適用されません（132条の11第3項）。この場合は書面提出が可能です。障害発生時はスクリーンショット等で状況を記録しておくことを推奨します。
✎ 義務違反の効果について 代理人が義務に反して書面提出した場合の取扱い（不適法却下・ファイル不記録等）について、公式資料での明確な記載は確認できていません（解説記事レベルでは「書記官がファイルに記録しない」等の指摘あり・二次情報）。少なくとも実務上、新法事件で代理人が就いている限り書面提出は受け付けられない前提で対応してください。

1.2 新法事件と旧法事件の区別

義務化は施行日（令和8年5月21日）以降に提起された訴えに適用されます。施行前に提起された事件（旧法事件）は、引き続き書面により審理・送達が行われます（裁判所公式）。当面は両者が併存するため、事件ごとに「新法事件か旧法事件か」を最初に確認する運用が必須です。

新法事件（R8.5.21以降提起）旧法事件（R8.5.20以前提起）
書面提出代理人は mints 義務。mints上の電子データ自体が訴訟記録従来どおり書面・FAX（mintsは従前の任意利用の範囲）
送達システム送達（1週間みなしあり）従来どおりの送達
手数料ペイジー電子納付（印紙・郵券廃止）収入印紙・郵券。mintsの納付機能は使用厳禁
受領書不要（アクセス記録が代替）必要（mints上で受領書作成・提出）

1.3 システム送達と「1週間みなし送達」

新法事件では、裁判所からの判決書・期日呼出状等は mints へのアップロード＋通知メールによって送達されます（民訴法109条の2）。送達の効力は次のいずれか早い時に生じます（109条の3）。

  1. 送達を受けるべき者が mints 上でファイルを閲覧した時
  2. 送達を受けるべき者がファイルをダウンロード（記録）した時
  3. 通知が発せられた日（弁護士等の義務対象者は閲覧等の措置がとられた日）から1週間を経過した時
🚨 最重要 ― 期限管理
  • 判決書がアップロードされると、閲覧しなくても1週間後に送達が成立し、そこから控訴期間（2週間）が起算されます。例：4月1日アップロード → 未閲覧でも4月8日に送達成立 → 控訴期限は4月22日。
  • 弁護士等の義務対象者には、届出をしていなくてもシステム送達ができ、通知メールの発出すら法律上は不要とされています（109条の4）。メールが届かなくても送達は進行し得るため、係属事件がある間は mints へ定期的（最低週1回以上推奨）にサインインして確認してください。
  • 通知メール（info@mints.courts.go.jp 等）が迷惑メールに振り分けられないよう受信設定を必ず行うこと（→2章）。

1.4 手数料の電子納付・郵券廃止

  • 新法事件の申立手数料は、収入印紙に代わりペイジー（Pay-easy）による電子納付になりました（操作は第7章）。
  • 郵券（予納郵便切手）の予納は廃止され、送達費用は申立手数料に一本化されました。
  • 納付を怠ると、補正依頼 → 裁判所書記官の納付処分を経て、裁判長による訴状却下命令に至ることがあります（民訴法137条の2参照・東弁LIBRA 2026年3月号）。

第3章初期設定

3.1 サインアップ（利用者登録） 公式マニュアル p.8〜13

裁判所に利用申請をすると（事件単位で書記官に申請、または招待キー方式）、「【mints】新規アカウント登録の御案内」メールが届きます。届いたら次の手順で登録します。

  1. Edge または Chrome で https://www.mints.courts.go.jp/user/ に接続し、「サインイン」→「今すぐサインアップ」を押下（サインイン画面は別ウィンドウで開く。ポップアップブロックに注意）
  2. メールアドレスを入力し「確認コードを送信」。招待メールを受信したアドレスのみ入力可
  3. 受信した確認コードを入力（有効期限は180秒。切れたら再発行）
  4. パスワード（英大文字・英小文字・数字・記号の4種類すべてを含む10〜64文字）と利用者情報を入力し「Create」
  5. 二要素認証：電話番号を入力し、SMS「コードの送信」または「電話する」（自動音声）で認証
  6. 「アカウント登録完了」メールが届けば完了。メールに記載されるID（14桁）は今後裁判所に伝える場面があるため控えておく
⚠ 入力時の注意
  • 氏名・住所は全角で入力（氏名カナは全角カタカナ）。
  • 氏名・生年月日・住所は登録後に自分で変更できません（変更には証明書類のアップロードと裁判所への申請が必要）。誤字に注意。
  • 途中で「キャンセル」を押すと入力データは保存されません。

3.2 弁護士の登録注意点

  • 住所欄には自宅でなく事務所住所を登録し、末尾に「（送達場所）」と入力する（公式マニュアルの指示）。事務所名の記載も推奨（夕陽ヶ丘法律事務所ブログ）。
  • 登録情報は新規申立てフォームの当事者情報入力時に「入力者の情報呼出」で自動転記されるため、ここが誤っていると訴状の表示がそのまま誤る。
  • 当事者ユーザのアカウントは1人1つのみ（複数アカウント不可）。
  • 士業者登録番号を入力しておくと、1年間未サインインでも自動削除されない（未入力の一般ユーザは最終サインインから1年で自動削除）。

3.3 本人・資格確認（改正法施行後は必須） 公式マニュアル p.22〜26

  • 改正法施行後は、新規申立てや招待キー入力等の全機能を使うために本人・資格確認が必要。裁判所側で本人確認フラグが「済」になると「新規申立一覧」「事件一覧」「手数料納付情報一覧」「招待キー入力」が有効化される。
  • 弁護士の本人確認書類：所属弁護士会の身分証明書＋単位会発行の印鑑証明書のPDF（アカウント設定画面の「本人・資格確認資料」欄にアップロード）。窓口での本人確認も可。
  • 経過措置：令和8年5月20日以前に登録済みのアカウントはそのまま利用でき、本人確認書類の再提出は不要（夕陽ヶ丘法律事務所ブログ）。
🚨 マイナンバーカードをアップロードしない 本人確認資料としてマイナンバーカードを mints にアップロードしてはいけません（裁判所公式の注意喚起）。誤ってアップロードした場合は直ちに裁判所へ連絡してください。

3.4 補助者（事務職員）の設定 公式マニュアル p.27〜28

弁護士（親ユーザ）は、事務職員を補助者ユーザとして関連付けることができます。補助者の操作はすべて親ユーザが行ったものとみなされ、閲覧・ダウンロード等のアクセス記録も親ユーザ名義で付されます。

ルール内容
設定可能数1人の親ユーザにつき補助者IDは5つまで
兼任同一の補助者アカウントを複数の弁護士が共有することは不可。複数の弁護士の補助者になる職員は、弁護士ごとに別アカウントを取得（1人10アカウントまで）
できること事件情報の確認／新規申立て／アップロード／閲覧・DL／印刷／受領書作成／納付情報確認（いずれも親ユーザ名義）
できないこと招待キーの入力（親ユーザ本人の操作が必要）

設定手順：① 裁判所に補助者となる職員のメールアドレスを申請 → ② 職員宛に招待メールが届き、職員がサインアップ（氏名欄は「補助者△△△△」形式） → ③ 職員がアカウント設定画面で自分のID（14桁）を確認して弁護士に伝達 → ④ 弁護士が自分のアカウント設定画面で補助者IDを登録。

⚠ 補助者運用の注意
  • 職員が離職した場合は、必ず関連付けを解除する（アカウント設定画面で対象の補助者IDを削除して更新）。
  • 解除の際、「アカウント削除」ボタンを押すと自分（親ユーザ）のアカウントが消えるので絶対に間違えないこと。

第4章サインインとパスワード 公式マニュアル p.14〜20

  1. https://www.mints.courts.go.jp/user/ →「サインイン」を押下
  2. メールアドレスとパスワードを入力
  3. 二要素認証：SMS「コードの送信」（コード有効期限180秒）または「電話する」（自動音声）
  4. ホーム画面が表示されたら完了

パスワードを忘れた・変更したいとき

サインイン画面の「パスワードを忘れた場合」から、①登録メールアドレスに確認コード送信 → ②コード入力 → ③二要素認証 → ④新パスワード設定、の順で再設定できます。パスワード要件は英大文字・英小文字・数字・記号の4種類すべてを含む10〜64文字。メールアドレスは大文字・小文字も正確に入力してください。登録メールアドレス自体が不明な場合は係属裁判所へ問い合わせます。

⚠ アカウント設定変更時の注意 二要素認証用電話番号やメールアドレスを誤って更新すると、次回サインインできなくなり、通知メールも届かなくなります。変更時は入力内容を十分確認してください。過去に別アカウントで使用したメールアドレスへの変更は「サーバ処理が失敗しました」エラーになることがあります（その場合は問い合わせフォームへ）。

第5章日常操作（事件確認・提出・受領）

5.1 ホーム画面とお知らせ 公式マニュアル p.21, 60〜62

  • 「重要なお知らせ」「新着情報」には提出期限・アップロード・印刷データ生成等の通知が掲載日から2週間表示され、その後さらに2週間は「過去のお知らせ」から閲覧可能（リンクは消える）。
  • サイドバー「事件一覧」→ 事件の行をクリックで事件情報画面へ。提出期限が設定されたファイル、提出済みファイル等を確認できる。
  • 事件情報画面のタブ：記録一覧（主張・証拠・証拠説明書・調書・裁判書等＝訴訟記録）／記録外一覧／関連事件一覧。

  • 対応形式外のデータ（特殊な音声・動画等）は従来どおりCD-R等の媒体で裁判所・相手方に提出（「フェーズ3に向けた準備の手引」28頁）

手順

  1. 事件情報画面で「提出ファイル選択」から対象を選ぶ。提出期限が設定されたファイル（「準備書面」等）はその行を選択、それ以外は「提出期限のないファイル」を選択して「アップロード」を押下
  2. ファイル種別タブ（主張／証拠／証拠説明書／その他／記録外 等）を選択し、ファイルを選択またはドラッグ＆ドロップ。複数選択可だが順番どおりに並ばないことがあるため、順序が重要なら1ファイルずつ
  3. 「追加」→ 一覧を確認 → 「提出」を押下（チェックボックスにチェックが入っていると提出ボタンが押せない）
  4. 確認メッセージで「OK」→ ウイルススキャン（数分）→ 完了すると新着情報に通知され、全当事者にメールが送信される
  5. 証拠の場合は「証拠番号を編集」で 甲001 乙002 のように半角3桁で入力（記録一覧の並べ替えに使われる。主張書面は入力不要）
🚨 アップロードは取り消せない 提出後のファイルの差替え・削除・ファイル名や書面種別の変更は当事者側ではできません。誤って提出した場合は担当書記官に電話して削除を依頼します。「提出」を押す前にプレビューで中身を必ず確認してください（アップロード画面のファイル選択時にプレビューウィンドウで内容確認可）。
✎ 書証ファイルの作り方（公式FAQの推奨）
  • 1書証1ファイルが原則。枝番のある書証は「甲001_1〜3」のように1ファイルにまとめてよい。
  • 書証データの右上に証拠番号を記載しておく。
  • 提出するPDFには裁判所側でタイムスタンプが付され、これが受付印の代わりになる。
  • 提出するのは「写しに代わる画像情報（PDF）」だが、取調べには期日に原本の持参が必要（→11.5）。
  • 証拠説明書は公式の【新法適用事件用】証拠説明書（Excel）を用いる（書証＝甲乙号証と電磁的記録の証拠説明書を兼ね、証拠番号は通番で付す）。書き方は証拠説明書 記載要領（PDF）参照。→ 第13章書式集

5.3 ダウンロード 公式マニュアル p.70〜73

  • 単一ファイル：事件情報画面でファイルを1つ選択 →「ダウンロード」。
  • 複数ファイル：複数選択 →「ダウンロード」→ データ生成後、ホームの「重要なお知らせ」の「ダウンロードデータの生成が完了しました。」を押下すると開始。ZIP形式でまとまる。
  • 結合ダウンロード：複数選択 →「一つのファイルにしてダウンロード」で1本のPDFに結合できる（記録の通し読みに便利）。
  • ZIPはWindows標準機能で解凍する（他の解凍ソフトはファイル名が文字化けすることがある）。
  • 生成・ダウンロードには数分かかることがある。

5.4 印刷 公式マニュアル p.74〜76, 92〜94

  • 印刷機能で印刷できるのはPDFのみ（JPEG・PNGはダウンロードして印刷）。
  • 単一・複数とも「一括印刷」ボタンから。複数の場合は印刷データ生成後、「重要なお知らせ」から印刷プレビューを開く。
  • A3とA4が混在する場合はプレビューが出ず、PDFがダウンロードされるので開いて印刷する。
  • 未印刷物一括印刷：事件一覧画面の「未印刷物一括印刷」→「開始」で、未印刷のファイルをまとめて印刷できる。印刷済みのファイルは対象から外れる。300MB超は分割生成。
  • Adobe Acrobat Reader の印刷設定で「PDFのページサイズに合わせて用紙を選択」にチェックを入れておくと、A3/A4が原寸で出力される（→第9章）。

5.5 受領書（旧法事件のみ） 公式マニュアル p.77〜79

  • 旧法事件では、相手方からファイルがアップロードされたら受領書を提出する（新法事件と記録外ファイルでは不要）。
  • 手順：対象ファイルを1つだけ選択 →「受領書作成」→ 内容確認（必要なら編集）→「受領書提出」。同時にアップロードされたファイルがまとめて対象になる。
  • 2つ以上選択すると受領書作成ボタンが押せなくなる点に注意。

5.6 記録外ファイル 公式マニュアル p.80〜82

  • 「記録外」は正式な提出にはならない参考資料の置き場（Word・Excel等も可）。判決書の修正用データや事務連絡等に使われる。
  • 記録外にアップロードしたファイルも、事件に関連付けられた全当事者が閲覧できる。相手方に見せたくない資料を置かないこと。

5.7 アクセス記録は相手に通知される 公式マニュアル p.63, 別紙1

⚠ 知っておくべき仕様
  • ファイルを初めて閲覧・ダウンロード・印刷すると「アクセス状況の記録」が付き、事件に関連付けられた全当事者・補助者にメールで通知されます（「【mints】ファイルが閲覧・ダウンロードされました。」）。つまり、こちらが相手方の書面をいつ閲覧したかは相手方にも分かります（逆も同様）。
  • 補助者が閲覧した場合も親ユーザ（弁護士）名義で記録されます。
  • 新法事件では、このアクセス記録が送達効力の発生（109条の3第1項1号・2号）や受領書の代替として機能します。

第6章新規申立て（オンライン提訴） 公式マニュアル p.30〜55

新法事件の訴え提起・控訴・上告・支払督促等は、サイドバー「新規申立一覧」→「新規作成」のフォームから行います。

6.1 全体の流れ

  1. 入力者肩書・申立種別を選択（訴え提起／控訴／上告・上告受理／支払督促／その他）
  2. 当事者・代理人情報を入力（フォーム入力は合計10人まで。超える場合はCSV提出が必須。公式の当事者情報CSVファイル作成ツールで作成しアップロードする→書式集）。「入力者の情報呼出」ボタンでアカウント登録情報を自動転記できる
  3. 申立内容を入力：提出先裁判所・事件種別・事件名・被告の数・訴額・申立ての趣旨（400字以内）・申立ての理由（10,000字以内）
  4. 添付書類をアップロード：訴状本文のPDF・委任状・資格証明書等
  5. 参考事項を入力（訴え提起・控訴のとき表示。相手方には開示されない）
  6. 「保存」で一時保存 →「プレビュー」「申立ダウンロード」で申立書の仕上がりを確認 →「提出」
  7. ステータスが「提出中」→「提出済」になり、受付番号・受理日が表示される。完了メールも届く。ただしここで手続が終わるわけではない。この後、裁判所による立件（事件番号付与）→ 代理人の事件への関連付け → 追加書面の提出 → Teams登録 → 手数料納付 → 期日調整、という一連の流れが続く（→6.3）

  • 一時保存の保存期限は最終保存日から1か月。期限内に保存も提出もしないと削除される。
  • 「保存」しただけでは提出になっていない（「提出ボタンをクリックするまで提出は完了となりません」）。
✎ 実務の段取り（夕陽ヶ丘法律事務所ブログ）
  • 新規申立て時に添付するのは訴状・委任状・資格証明等のPDF。証拠・証拠説明書は裁判所の事件登録完了後に、事件情報画面からアップロードする。
  • 被告に代理人が付いていない場合、被告への訴状送達は原告代理人が提出する出力書面（訴状を印刷したもの）で行われる運用が多い（改正民訴規則58条1項参照）。裁判所からの指示に従って準備する。
  • 簡裁への同種大量申立て（債権回収等）には「簡裁用一括申立て」機能があり、当事者情報等をCSVで提出する。

6.3 提出後の流れ ― 立件・関連付け・Teams・第1回期日

訴状をmintsで「提出済」にしても、そこで手続が完結するわけではありません。実際には、受訴裁判所（担当書記官）からの案内（mintsの通知・電話・FAX等）に沿って、次の一連の作業が続きます。以下は当事務所が実際に受けた案内に基づく典型的な流れです（具体的な提出物・順序・呼称は裁判所・事件・係属部により異なるため、必ず個別の案内を最優先してください）。

  1. 裁判所が事件情報を作成：裁判所が訴状を確認して立件し、mints上に事件情報を作成する。作成されると通知メールが届く（提出から概ね1〜2時間で事件番号が付与された例あり・二次情報→11.6）。まずmintsでこの事件情報を確認する。
  2. 代理人を事件情報に「関連付け」する届出をアップロード：立件された事件情報に代理人を紐付けるため、裁判所の指定に従い、関連付けの範囲に応じて次のいずれかをmintsにアップロードする。
    • 入力者の弁護士のみを関連付ける場合 → 電子情報処理組織の使用を担当する訴訟代理人の届出（規則52条の13）
    • 共同代理人全員を関連付ける場合 → 代理人全員のシステム送達を受ける旨の届出（民訴法109条の2・規則45条の3）
  3. 入力者（担当弁護士）自身の追加提出：あわせて、① 入力者のシステム送達を受ける旨の届出、② 証拠説明書、③ 甲号証（書証PDF） を事件情報画面からアップロードするよう求められる（証拠は立件後に出す＝11.5）。
  4. Teams（ウェブ会議）の登録：期日はMicrosoft Teamsのウェブ会議で行われるため、裁判所がTeamsのチームを作成し、指定アドレスへメール送信するよう案内される。メールには通常、① 登録するメールアドレス（Microsoftアカウントがあればそのアドレス）、② 民事訴訟手続でのTeams利用経験の有無、③ 第1回期日の出頭方法（リアル出頭／ウェブ出頭） を記載する。
  5. 訴状審査 → 手数料納付依頼：訴状審査後、裁判所から手数料の納付依頼が届く。ペイジーで納付する（→第7章）。
  6. 手数料納付後 → 期日調整：納付確認後、Teams上で第1回期日等の日程調整を行う。
  7. 被告送達用の出力書面を郵送等で提出：被告に代理人が付いていない等の場合、被告への送達用に訴状等を印刷した出力書面を郵送等で裁判所に提出する（改正民訴規則58条1項参照）。
✎ 押さえどころ
  • 「提出済」＝完了ではない。立件通知・Teams招待・納付依頼・期日調整はチャネルが分かれるため、mintsの通知メールとTeamsの双方をこまめに確認する。
  • 新規申立てフォームでシステム送達届出をONにしていても、裁判所が改めて書面（届出のdocx）のアップロードを求めることがある。案内された書式を出す。
  • 関連付けの範囲（担当弁護士のみ／代理人全員）で提出書式が変わる。共同受任なら早めに方針を決める。
  • 提出後に求められやすい書式は第13章の書式・様式ダウンロード集からまとめて入手できる（受任時にテンプレ化しておくと段取りが速い）。
⚠ 出典と留意 提出後の具体的手順（関連付けの届出・Teams登録・出力書面の郵送等）は受訴裁判所からの個別案内に基づく実務であり、裁判所・事件・係属部によって運用が異なります。Teamsによるウェブ会議・期日調整が用いられること自体は公式資料・複数の実務解説でも確認できます（→第14章）。本節は一般化した流れであり、必ず担当書記官の案内を最優先してください。

第7章手数料納付（ペイジー） 公式マニュアル p.56〜59

🚨 旧法事件では使用厳禁 手数料納付機能が使えるのは新法事件（改正法適用事件）のみです。旧法事件では mints による手数料納付は絶対にしないでください（公式マニュアルの強い注意書き）。
  1. 裁判所が手数料納付情報を登録すると、納付義務者（と補助者）に通知メールが届き、ホームの「重要なお知らせ」にも表示される
  2. サイドバー「手数料納付情報一覧」で、ペイジーの「収納機関番号」「納付番号」「確認番号」を確認（番号にカーソルを合わせるとコピー可）
  3. インターネットバンキングまたはATM（ペイジー対応）で納付する
  4. 納付が完了すると一覧の「納付日」に日付が入り、「新着情報」にも通知される
  • 納付期限の14日前・7日前・2日前・1日前・当日にリマインドメールが届く。
  • 納付期限を過ぎると、その納付情報では納付できなくなる。失念したら速やかに裁判所へ連絡。放置すると納付処分 → 訴状却下命令（民訴法137条の2）に至り得る。
✎ 番号の控え方 mints には納付番号を印刷する機能がないため、画面のスクリーンショットで番号を控えて経理に回す運用が現実的です（夕陽ヶ丘法律事務所ブログ）。

第8章招待キー・事件アクセスキー 公式マニュアル p.84〜91

8.1 招待キー（当事者・代理人用）

  • 裁判所から交付される12桁の半角英数字。入力すると、書記官の関連付け作業を待たずに自分で事件の当事者に関連付けられる。
  • 操作：サイドバー「招待キー入力」→ キーを入力 →「送信」。メールで受領した場合はホームのお知らせからも遷移可。
  • 補助者ユーザは入力できない。親ユーザ（弁護士本人）が操作する。
  • 招待キーには有効期限がある（メールに記載）。「使用できません」エラーの場合は係属部に問い合わせる。

8.2 事件アクセスキー（第三者用）

  • 事件の当事者ではない第三者（文書送付嘱託の嘱託先金融機関等）がファイルをアップロードするための12桁キー。
  • 第三者は事件に関連付けられないまま、「事件アクセスキー入力」からアップロードのみ行える。

第9章PDF作成・ファイル変換のコツ 公式マニュアル 別紙3〜5

9.1 Word/Excel → PDF 変換（メタデータを残さない方法）

🚨 作成者名などの個人情報に注意 通常の方法でPDF化すると、ファイルのプロパティに作成者名等が残り、相手方がダウンロードしたPDFからそれが見えることがあります。PDF化した後ではプロパティ・個人情報の削除はできません。必ず下記オプション設定で変換してください（DV・ストーカー関連事件等では特に重要）。
  1. Word/Excelで「ファイル」→「名前を付けて保存」→ ファイル形式「PDF(*.pdf)」を選択
  2. 「その他のオプション」→「オプション」を開く
  3. 次のチェックをすべて外す：「PDF/A準拠」／「ドキュメントのプロパティ」／「アクセシビリティ用のドキュメント構造タグ」／「フォントの埋め込みが不可能な場合はテキストをビットマップに変換する」
  4. 保存し、できたPDFのプロパティで作成者等が消えていることを確認

Word/Excelファイル自体（記録外で提出する場合）のプロパティ削除は、ファイルを右クリック →「プロパティ」→「詳細」タブ →「プロパティや個人情報を削除」→「可能なすべてのプロパティを削除してコピーを作成」で行えます。

9.2 用紙サイズの修正（A4/A3以外はエラーになる）

  • Word/Excel：「レイアウト」タブ →「サイズ」でA4またはA3に変更してからPDF化。
  • 既存PDF：「印刷」→ プリンターに「Microsoft Print to PDF」を選択 → プロパティの詳細設定で用紙サイズをA4/A3に指定して出力し直す。
  • スキャナーで書面をPDF化する場合も、出力サイズがA4/A3になっているか確認する。
✎ パスワード付きPDFの解除も同じ方法 パスワード保護されたPDFはアップロードできません。パスワードを知っている場合は、開いた状態で「Microsoft Print to PDF」で出力し直せば保護のない別ファイルが作れます。

9.3 PDFの向き

  • 縦置きの書面は縦向き、横置きの書面は横向きのPDFでアップロードすることを推奨（タイムスタンプが適切な位置に付されるため。公式マニュアル別紙4）。
  • 横向き → 縦向きの変更も「Microsoft Print to PDF」での出力し直しで可能。

9.4 Acrobat Reader の印刷・閲覧設定（最初に1回だけ）

  • 設定1：印刷ウィンドウの「ページサイズ処理」で「PDFのページサイズに合わせて用紙を選択」にチェック。A3/A4混在の記録が原寸で印刷されるようになる。一度設定すれば保存される。
  • 設定2：PDFの既定アプリをAcrobat Readerにし、ブラウザのダウンロード通知で「常にシステムビューアーで開く」にチェック。ダウンロードしたPDFが自動で開くようになる。

「ファイルの形式が正しくありません」対応形式（記録：PDF/MP3/MP4/JPEG/PNG）か確認。Word/Excelは記録には出せない → PDFに変換（第9章）
「ファイルの用紙サイズが正しくありません」PDFがA4/A3以外。Microsoft Print to PDF でサイズを直す（第9章）
「ファイルにパスワードが付されています」パスワードを解除して再アップロード（第9章）
「許容量（総ファイル容量）を超えたため、追加できません」1回200MBの上限超過。複数回に分けてアップロード
「ファイル名が重複しているため、提出できません」同名ファイルは同時に出せない。名前を変えるか、タイミングを分ける
「ファイル名を、100文字以下に設定してください」拡張子込み100文字以内に短縮
「{○○}に入力禁止文字が含まれています」外字・特殊文字（髙・﨑等）をJIS X 0213の代替文字に置換
「一定時間操作がなかったため、サインアウトしました」タイムアウト。再サインイン。新規申立てフォームの未保存分は消えているので注意
「本事件は係属中でないため、操作できません」終局した事件は操作不可。必要があれば書記官に連絡
「該当する招待キーがありません」「使用できません」入力ミスを確認。誤りがなければ係属部へ問い合わせ（有効期限切れの可能性）
「サーバ処理が失敗しました（エラー）」等再操作しても解決しない場合は、チャットボット「お問合せ窓口を知りたい」→問い合わせフォームへ

第11章事務所運用の実務Tips

11.1 体制づくり

  • mints専用メールアドレス（またはエイリアス）を用意し、通知メールの仕分けルールを整備する。通知は弁護士と補助者の双方に届くため、二重チェック体制にできる。
  • 補助者アカウントを設定し、アップロード・印刷等の定型作業を事務局に委ねる（操作はすべて弁護士名義になる点は事務局にも周知）。
  • 係属事件がある間は、メール頼みにせず定期的にサインインして「重要なお知らせ」を確認する習慣を付ける（1週間みなし送達対策）。

11.2 依頼者との関係

⚠ 依頼者本人に mints アカウントを作らせない 代理人が就いている事件で依頼者本人もユーザ登録すると、依頼者が判決書を閲覧した時点で送達が成立し、そこから上訴期間が起算されるなど、送達・期限管理が事務所のコントロール外になります。記録の共有は、事務所でダウンロードしたデータを別途交付する方法によるのが安全です（夕陽ヶ丘法律事務所ブログ）。

11.3 提出時の注意

  • 期限ぎりぎりの深夜提出は避ける。毎月最終土曜午前2時〜10時の定期メンテナンスのほか、障害リスクもある。アップロード後のウイルススキャンにも数分かかる。
  • 相手方の書面を開くタイミングは相手に通知される（5.7）。戦略的に閲覧時期を考える場面があり得る一方、送達対象ファイルは放置しても1週間で効力が生じる。
  • 書証はmintsに「写しに代わる画像情報（PDF）」を提出するが、取調べには期日に原本の持参が必要で、契約書等は原本の書証申出が望ましい（改正民訴規則137条・民訴規則143条。詳細と根拠は11.5）。裁判体の訴訟指揮に従う。
  • 補正依頼は mints 経由のこともあれば従来どおり電話のこともある（東弁LIBRA）。

11.5 書証・証拠提出の規律（改正民訴規則） フェーズ3準備の手引 R8.6.19

新法事件では、証拠の提出方法についても改正民訴規則（民事訴訟規則等の一部を改正する規則＝令和8年最高裁判所規則第7号による改正後のもの）で規律が整理されました。「mintsにPDFを出す」ことと「書証の取調べ」は別である点が最重要です。

📖 書証の写しに代わる画像情報の提出／原本の持参
訴訟代理人は、文書を提出して書証の申出をする時までに、……書証の写しに代わる画像情報（PDF形式）をmintsにアップロードして提出しなければならない（改正民訴規則137条）。
書証の申出は、文書を提出することによってしなければならず（民訴法219条）、文書の提出は、原本、正本又は認証のある謄本でしなければならないという規律に変わりはないことから（民訴規則143条）、書証を取り調べるためには、期日に書証の原本を持参する必要がある。
出典：最高裁判所事務総局民事局「民事訴訟フェーズ3に向けた準備の手引」R8.6.19版（一次情報）
  • 提出はPDF、取調べは原本：mintsには「写しに代わる画像情報（PDF）」をアップロードするが、これで取調べが完結するわけではなく、期日には書証の原本を持参する。契約書等、原本の存在・成立の真正が争われ得る類型では特に、原本による書証申出が望ましい。
  • 電磁的記録としての証拠調べ：紙の証拠をPDF化し「電磁的記録に記録された情報の内容に係る証拠調べ」（改正法で新設）として申し出ることも可能。ただし相手方が真の原本の存在・成立を争い実質的証拠力に疑義が生じると、裁判所が改めて真の原本の書証申出（新たな証拠番号を付す）を訴訟指揮することがあり、合理的理由なく原本申出をしないと実質的証拠力が慎重に検討される。
  • 証拠説明書：電子証拠説明書と兼ねて作成し電子提出する。書証と電磁的記録の各証拠番号は通番で付す。
  • アップロード時期：証拠は新規申立て時ではなく、裁判所が事件を立件した後に事件情報画面からアップロードする（→第6章のTips）。
✎ ウェブ尋問・書証の提示（同手引） ウェブ尋問の要件が見直され、証人等の出頭場所が裁判所以外にも拡大されました。尋問で文書等を証人に示す場合は、紙媒体を提示する方法のほか、データを画面共有等で提示する方法によります。運用は裁判体の訴訟指揮に従ってください。
